# Build Release — Vyntra Windows Installer

## Prerequisites

- Rust toolchain with `x86_64-pc-windows-msvc` target
- Node.js + pnpm
- NSIS (installed automatically by Tauri if missing, or install manually from https://nsis.sourceforge.io)

## Steps

### 1. Install dependencies (first time or after pnpm-lock.yaml changes)

```
pnpm install
```

### 2. Build the release installer

```
cd apps/desktop
pnpm build:vite          # Build the Vite frontend (outputs to apps/desktop/dist/)
pnpm tauri build         # Compile Rust + bundle NSIS installer
```

Or from the repo root (if a root-level build script is added):

```
pnpm --filter @vyntra/desktop tauri build
```

### 3. Output

The installer is produced at:

```
target/release/bundle/nsis/Vyntra_0.1.0_x64-setup.exe
```

The version in the filename matches `version` in `tauri.conf.json` (and `Cargo.toml`).

## Before shipping

- [ ] Replace `REPLACE_WITH_REAL_MINISIGN_PUBKEY` in `tauri.conf.json` with the actual Minisign public key generated by `tauri signer generate`
- [ ] Replace `REPLACE_WITH_UPDATE_ENDPOINT` in `tauri.conf.json` with the real update server URL (e.g. `https://releases.vyntra.app/{{target}}-{{arch}}/{{current_version}}`)
- [ ] Sign the installer with a code-signing certificate to avoid Windows SmartScreen warnings

## Generating a Minisign keypair (one-time setup)

```
pnpm tauri signer generate -w ~/.tauri/vyntra.key
```

The public key printed to stdout goes into `tauri.conf.json` → `plugins.updater.pubkey`.
The private key (`vyntra.key`) is used to sign update artifacts — keep it secret.

## Notes

- `installMode: "currentUser"` — installs to `%LOCALAPPDATA%\Programs\Vyntra`, no admin rights required.
- Dev mode (`pnpm dev` from repo root) is unaffected by these bundle settings.
- WiX target is intentionally excluded; NSIS is the only bundler configured.
